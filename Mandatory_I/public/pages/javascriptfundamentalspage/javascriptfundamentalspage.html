1. Basics
This section covers some of the basics of JavaScript. We will have a look at variable declaration and the console. There will be an example of the difference between using template literals and comma-separated values in the console.log() function.
Console and Variables
This example and these code snippets emphasize the difference between using literals and comma-separated values in the console.log() function. The first example uses template literals, which are enclosed in backticks (`). The second example uses comma-separated values, which are separated by commas (,). The output of both examples is the same, but the template literals are easier to read and write. 


    const firstName = "Anders";
    const lastName = "Latif";
                
    // Using template literals
    console.log(`My first name is ${firstName} and my last name is ${lastName}`)
    // Output: My first name is Anders and my last name is Latif
                
    // Using comma-separated values
    console.log("My first name is", firstName,"and my last name is", lastName)
    // Output: My first name is Anders and my last name is Latif
Numbers and Strings
In this example the importance of type conversion is shown and also different ways to convert a string into a number. It also showcase how Number() and parseInt() handles non-numeric characters differently. 


    const year = "2022";
    const number = 1;
    
    // Convert string to a number and then add
    const result1 = Number(year) + number;  // Using Number()
    console.log(result1);  // Output: 2023
    
    const result2 = parseInt(year) + number;  // Using parseInt()
    console.log(result2);  // Output: 2023
    
    const result3 = +year + number;  // Using the unary + operator
    console.log(result3);  // Output: 2023
    
    // Example of how Number() and parseInt() handle non-numeric characters
    console.log(Number("32566dsf6546"))  // Output: NaN
    console.log(parseInt("659746ad644"))  // Output: 659746

5. Functions
In JavaScript, functions can be defined in many ways. In this section, I will show you how to create a function, how to call a function and how to pass arguments to a function.
Function Hoisting
In JavaScript, function declarations are hoisted, allowing them to be called before they appear in the code. This example illustrates this concept of hoisting. Additionally, the example differentiates between logging a function reference versus its return value when no arguments are provided. Not supplying the expected arguments can often lead to unexpected outcomes, such as returning `NaN`. 


    console.log(getRandomInt()); // Output: NaN (because no arguments are provided)

    function getRandomInt(min, max) {
        return Math.floor(Math.random() * (max + 1 - min) + min);
    }
    
    console.log(getRandomInt); // Output: [Function: getRandomInt]
    console.log(getRandomInt()); // Output: NaN (no arguments)
Different Function Definitions
This example demonstrates the different ways to define a function. The first example is an anonymous Function which assign an anonymous function to a constant, the second example is an arrow function with return were the body is enclosed in {} and therefore requires an explicit return and the third example is an arrow function without return and a concise body, there are no curly braces, and the function's result is directly returned without the return keyword. This is a shorthand that makes arrow functions concise for single-expression functions, the last one is a way to add the result of a function directly to a variable. 


    const getRandomIntAnonymousFunction = function (min, max) {
        return Math.floor(Math.random() * (max + 1 - min) + min);
    }
    
    const getRandomIntArrowFunction = (min, max) => {
        return Math.floor(Math.random() * (max + 1 - min) + min);
    }
    
    const getRandomIntArrowFunctionWithoutReturn = (min, max) => 
        Math.floor(Math.random() * (max + 1 - min) + min);
    
    const result = getRandomIntArrowFunction(1, 100);
    console.log(result); // Output: Random number between 1 and 100
Using Callbacks
This example demonstrates how to use callbacks. A callback is a function passed as an argument to another function. This technique allows a function to call another function. 


    function sayHello(callback) {
        console.log("Hi, this is before callback!");
        callback();
    }
        
    function myCallback() {
        console.log("Hi, this is inside the callback!");
    }
        
    sayHello(myCallback); 
    // Output: 
    // Hi, this is before callback!
    // Hi, this is inside the callback!
Generic Function with Callback
This example demonstrates how to use a generic function with a callback. The generic function is called genericActionPerformer and it takes two arguments, a genericAction and a name. The genericAction is a callback function that takes a name as an argument and returns a string. The genericActionPerformer function returns the result of the genericAction function. 


    function genericActionPerformer(genericAction, name) {
        return genericAction(name);
    }
        
    const jump = (name) => `${name} is jumping`;
    console.log(genericActionPerformer(jump, "Lasse")); // Output: Lasse is jumping
    
    const run = (name) => `${name} is running`;
    console.log(genericActionPerformer(run, "Jonathan")); // Output: Jonathan is running
    
    function run2(name) {
        return `${name} is running`;
    }
    console.log(genericActionPerformer(run2, "Thomas")); // Output: Thomas is running
    
    // Desired result using arrow function:
    console.log(genericActionPerformer((name) => `${name} is sleeping`, "Daniel")); // Output: Daniel is sleeping

In JavaScript, there are many methods available to perform loops over arrays or lists. Among them there are methods like, 'forEach', 'map', 'filter', 'find', 'sort' and 'reduce'. These methods uses the principles of functional programming, which is allowing you to work with data in a declarative way.
Trolls example
Here we have a list of trolls, this list is an array of objects. Each object has a name and a trollLevel property. Throughout this example i will show how you can modify with some different methods. 


    const trolls = [
        { name: 'anonymous', trollLevel: 9 },
        { name: 'kroelleboelle', trollLevel: 22 },
        { name: 'kroelleline', trollLevel: 27 },
    ];
Using map() to create a upgraded List
This example demonstrates how you can change values in an array of objects. The map() method creates a new array populated with the results of calling a provided function on every element in the calling array. 


    const upgradedTrolls = trolls.map(troll => {
        troll.trollLevel += 5;
        return troll;
    });   
        
    // does the same as above, here is multiple expressions, but the last one is the one returned.
    const upgradedTrolls = trolls.map(troll => ((troll.trollLevel += 5), troll));

    //Another kind of syntax
    //Here we return an javascript object by encapsuling the the object with ()
    const doubleUpgradedTrolls = trolls.map(troll => ({
        name: troll.name,
        trollLevel: (troll.trollLevel += 5),
    }));

This example demonstrates how to use filter() to create a new list with trolls with an odd troll level.

When to use what
This example will show when to use what. Map, filter, reduce and find. I will also show how to use the sort() method. All of this is a part of how to make functional programming.
Map is used when you want to modify the data in the array.

In this example we have an array of numbers that we want to double. It will result in a new array with the doubled numbers. 


    //Array of numbers
    const numbers = [1, 2, 3, 4, 5];

    //The new array with the manipulated data
    const doubled = numbers.map(number => number * 2);

    console.log(doubled);
    //Output: [2, 4, 6, 8, 10]
Filter is used when you want to filter out data from the array.

In this example we have an array of numbers that we want to filter out the odd numbers. It will result in a new array with the even numbers. 


    //Array of numbers
    const numbers = [1, 2, 3, 4, 5];

    //The new array with the manipulated data
    const evenNumbers = numbers.filter(number => number % 2 === 0);

    console.log(evenNumbers);
    //Output: [2, 4]
Reduce is used when you want to reduce the data in the array to a single value.

In this example we have an array of numbers that we want to reduce to a single value. It will result in a single value. 


    //Array of numbers
    const numbers = [1, 2, 3, 4, 5];

    //The new array with the manipulated data
    const singleValue = numbers.reduce((accumulator, currentValue) => accumulator + currentValue);

    console.log(singleValue);
    //Output: 15
Sort is used to sort the data in the array.

In this example we have an array of numbers that we want to sort. It will result in a sorted array. 


    //Array of numbers
    const randomNumbers = [5, 1, 3, 2, 4];

    //Here we sort the array
    randomNumbers.sort();
    
    console.log(randomNumbers);
    //Output: [1, 2, 3, 4, 5]

    //Array of names
    const randomNames = ["Jonathan", "Thomas", "Daniel", "Lasse", "Andreas"];

    //Here we sort the array
    randomNames.sort();

    console.log(randomNames);

Functional Programming
Functional Programming is a way to work with data in a declarative way. It is a way to work with data in a way that is more readable and easier to understand. It is a way to write code that is more maintainable and easier to debug. It also shows what you want to achieve instead of conventional loops that shows how you want to achieve it. Here is Filter, Map, Reduce and Find some good examples of functional programming. When you implement functional programming, make sure to keep it simple and handle one thing at a time. So you don't end up with some unwanted side effects. This is important because you want to make sure that the code is easy to read and understand and does what it is supposed to do. 


    //Simple function that shows how to make functional programming

    function doubleNumbers(numbers) {
        return numbers.map(num => num * 2);
    }
    
    const originalNumbers = [1, 2, 3];

    //You parse in an array of numbers and get a new array with the numbers doubled back.
    const doubled = doubleNumbers(originalNumbers);

    console.log(doubled);
    //Output: [2, 4, 6]
It keeps it very simple, you know what you want to achieve and it is easy to read and understand.

Variable Declarations and Scopes
In this example,we look into some of the key concepts related to variable declarations and scopes. Strict mode, which is a way yo ensure that the code is written in a way that is more secure and less filled work errors. Look into the differences between var, let, const. We also look into the difference between function and block scopes. We also look into variable shadows and how to avoid common mistakes with async code. 


    "use strict";  // This is for enforcing strict mode, which helps in catching common coding mistakes.

    // Avoid using global variables or 'var'. Var should only be used in legacy systems/code.

    var globalVariable = "This is an example of using var.";
    console.log(globalVariable);  // Output: "This is an example of using var."
    
    // 'const' declares a variable that can't be reassigned, but the content of an object or array can be modified.
    const isThisConstant = [];
    isThisConstant.push(1, 2, 3);
    console.log(isThisConstant);  // Output: [1, 2, 3]
    
    const isThisAlsoConstant = {};
    isThisAlsoConstant.name = "Jonathan";
    //It's possible to add key-value pairs to a constant object, you just can't reassign the object to another value.
    console.log(isThisAlsoConstant);  // Output: {name: "Jonathan"}
    
    // Function scopes are isolated and variables declared within cannot be accessed outside.
    function anotherScope() {
        // This is a function scope.
    }
    
    // Block scopes are defined by curly braces {}.
    {
        console.log("Hello, I'm in a block scope");  // Output: "Hello, I'm in a block scope"
    }
    
    // Variable shadows: Variables with the same name can exist in nested scopes without conflict.
    {
        let someVariable = true;
    
        {
        let someVariable = false;
        // This 'someVariable' is different from the outer block's 'someVariable'.
        }
    
        console.log(someVariable);  // Output: true
    }
    
    // Loops: Using 'let' in loops to avoid common mistakes with async code.
    for (let i = 0; i <= 5; i++) {
        setTimeout(() => {
        console.log(i);  // Outputs numbers 0 to 5, each after a second.
        }, 1000);
    }            
    
    //And not 'var'.

    for (var i = 0; i <= 5; i++) {
        setTimeout(() => {
          console.log(i);
        }, 1000);
