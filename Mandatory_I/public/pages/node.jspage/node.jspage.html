Node.js - setup
Node.js is and open-source, cross-platform runtime environment that will let you execute Javascript outside of a web browser. Built on Chrome's V8 Javascript engine and is used for developing server-side applications. Easy to scale and is used by many large companies such as Netflix, Uber, and Paypal. In Node.js there is also npm which is a package manager that will let you install and manage packages.

Let's Begin to build a Node.js project!
Create Directory for your project
To setup your project you will first have to find a suitable location for the new project. Once you have found a suitable location you can create a new directory for your project and cd to the directory. 


    #First create the directory for your project
    mkdir "my-project"

    #then cd into the directory
    cd .\my-project\

What is node_modules?
The node_modules directory is a central repository where Node.js dependencies (libraries, frameworks, plugins, and other code modules) are stored. When you install a package using the Node Package Manager (npm) or Yarn, the package and its dependencies are downloaded and placed into the node_modules directory of your project.

A module will normally consist of a folder with one or more files in it. The folder will be named the same as the module and will contain a package.json file which describes the module. .js file/files, README.md file, and a LICENSE file. But it can vary from module to module.
Nodemon
Nodemon is a tool that helps develop node.js based applications by automatically restarting the server when changes are being made, which is very useful when developing a node.js project. It also provides feedback about the status of the application and will let you know if there are any errors. Actually nodemon can also be used for other languages than node.js, but it is mostly used for node.js. Though nodemon is a good tool, its not recommended to use it in production, since it's a development tool, that takes up a lot of memory. 

    #To install nodemon in your project you will have to type in this line:
    npm install -g nodemon
    #The -g stands for global, which means that it will be installed globally on your computer.

    #If you just want to install nodemon as a development dependency, you can type in this line:
    npm install --save-dev nodemon

    #When using it in your project you will have to type in this line:
    nodemon [name_of_your_project_file].js
Node.js - Miscellaneous
Here we will go through subjects like, how to define and run scripts, how to define environment variables natively in node.js and how code and its position affects application memory and efficiency.
Define and run scripts
In node.js you can define scripts in your package.json file and run them in the terminal.

Here is an example of how to define and run scripts in node.js. 


    //In your package.json file you can define scripts like this.
    "scripts": {
        "test": "echo \"Error: no test specified\" && exit 1",
        "start:dev": "cross-env PORT=8080 nodemon app.js",
        "start:prod": "cross-env PORT=80 node app.js"
    },
    //Here we have defined 3 scripts, test, start:dev, and start:prod.
    //start:dev and start:prod are both scripts that set the PORT environment variable and then runs the app.js file with nodemon or node.

            


    #To run the scripts you will have to type in this line in the terminal.
    npm run [name_of_script]

    #So if you want to run the start:dev script you will have to type in this line in the terminal.
    npm run start:dev
Set environment variables natively in node.js
To set environment variables natively in node.js you will have to use the process.env object.

Here is an example of how to set environment variables natively in node.js. 


    #First we have to set an environment variable in the terminal - in powershell its done like this.
    $env:DATABASE_URL = "mongodb://localhost:27017/DatabaseName"

            

    //You access the environment variables this way.
    const databaseUrl = process.env.DATABASE_URL;
    console.log(databaseUrl);
