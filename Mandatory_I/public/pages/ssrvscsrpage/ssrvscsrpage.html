SSR vs CSR
SSR or Server Side Rendering is a technique that renders the page on the server and sends the HTML to the client. The client then renders the page and displays it to the user.

CSR or Client Side Rendering is a technique that renders the page on the client. The client then renders the page and displays it to the user.
SSR - Server Side Rendering
This happens on the server, the server renders the page and sends the HTML to the client. The client then renders the page and displays it to the user. It's recognized by web crawlers and search engines, which makes it useful for SEO. It's also useful for the first page load, because the user doesn't have to wait for the page to load before seeing the content. The downside is that it's slower than CSR, because the server has to render the page and send the HTML to the client.
CSR - Client Side Rendering
Here the client renders the page and displays it to the user. It's faster than SSR, because the server doesn't have to render the page and send the HTML to the client. The downside is that it's slower for the first page load, because the user has to wait for the page to load before seeing the content. It's also not recognized by web crawlers and search engines, which makes it useless for SEO.
